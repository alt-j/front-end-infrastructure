Системы контроля версий

  • Больше не нужно помнить о всех исправленных ошибках и модификациях.
  • Нет смысла складировать ревизии на жестком диске.
  • Нет смысла переносить проекты вручную, достаточно получить их из репозитория.

Модульная система

  • Дробление сложных компонент на более мелкие.
  • Многократное использование функциональности (интерфейс).
  • Возможность обновления/замены модуля, без необходимости изменения остальной системы.

Инструменты сборки

  • Task runners (примеры: Grunt, Gulp).
  • Builders (примеры: webpack, enb).
  • JS minifier (примеры: UglifyJS, YUI Compressor).
  • CSS minifier (примеры: Clean-css, CSSSHRINK).
  • Images optimizer (примеры: Imgo, SVGO).

Тестирование

  • Еще один вариант документации.
  • Позволяет избегать глупых ошибок.
  • Обеспечаивает надежность кода.

Пример стека: Mocha + Chai + Sinon.JS + Karma runner

Непрерывная интеграция (CI)

Для продукта: частые релизы.
Для разработки: быстрое выявление интеграционных проблем.
